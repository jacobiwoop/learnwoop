C'est un document qui décrit ce que le produit doit faire, pourquoi, pour qui, et comment. C'est la bible du projet que tout le monde consulte : développeurs, designers, investisseurs.

Résumé du produit — C'est quoi, pour qui, pourquoi
Objectifs — Ce qu'on veut accomplir
Utilisateurs cibles — Les personas
Fonctionnalités — Ce que le produit fait (par rôle)
User stories — "En tant que prof, je veux... afin de..."
Structure de données — La BDD
Architecture technique — Les technos choisies
Maquettes — Les écrans
Roadmap — Le plan de développement
Critères de succès — Comment on sait que c'est réussi
💡 Pourquoi c'est important ?
Ça aligne tout le monde sur la même vision
Ça évite les malentendus avec les développeurs
Ça sert de référence tout au long du projet
Ça impressionne les investisseurs ou partenaires
